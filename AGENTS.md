# UltramanMath Agent Notes

本项目是给 5-6 岁孩子在手机横屏操作的纯 Web 前端 Phaser 2D 算术游戏。默认用中文沟通，先调查再改代码，页面类改动必须用浏览器截图或 DOM 方式验证。

## 项目边界

- 不使用 `frontend-design` 作为本项目的设计依据；这是儿童 2D 游戏，不是通用 Web 落地页或运营界面。
- 内部画布固定为 `960x540`，通过 Phaser Scale.FIT 适配手机横屏；关键 UI 必须在这个坐标系内完整显示，不依赖浏览器外层滚动。
- 数据只存本地 `localStorage`，存档键为 `ultramanmath-save-v1`；静音键为 `ultramanmath-muted`。不要引入后端、账号或联网存档。
- 音频使用 `public/assets/audio/theme.mp3` 作为菜单/Logo/一般界面音乐，`battle.mp3` 作为战斗音乐，按钮和战斗反馈使用本地短音效。
- 角色语音通过 `scripts/generate-mimo-voices.mjs` 生成到 `public/assets/audio/voices/`，运行时只加载本地 mp3，不在游戏里调用 TTS 接口。MiMo 本地网关生成时需要同时传 `api-key`、`Authorization: Bearer ...`，并显式设置 `stream: false`，否则可能只返回无音频的 SSE usage chunk；可用 `MIMO_VOICE_IDS=voice-hero-hurt` 这类逗号分隔 id 单条重生，避免已验收语音随机漂移。

## 素材与渲染规则

- `public/assets/generated/battle-backgrounds.png` 是 3 个横向排列的场景面板。取单个面板时必须用面板中心偏移公式：
  `imageX = WIDTH / 2 + (texture.width / 2 - (sceneIndex + 0.5) * panelWidth) * scale`。
  不要再写 `WIDTH / 2 - sceneIndex * panelWidth * scale`，那会把后两张背景移出视口，造成首页或故事模式像没有背景。
- 菜单、故事、挑战、商店和结算页都必须有可见背景；可以加暗色蒙层提升可读性，但不能把背景压到接近纯黑。
- 角色素材使用已清理透明底的 atlas。继续保留按 `列 x 行` 提取帧、按站立/攻击/绝招/受创生成纹理、底部落地锚点的逻辑；不要重新引入灰底、色键底或整图拉伸。
- 角色帧裁切时，贴左右边界、与主体水平断开、面积很小的连通块应视为相邻格串帧残留并剔除；雷德王站立帧曾因此露出右侧相邻帧尾巴。
- 左侧奥特曼 sprite 需要 `setFlipX(true)`，右侧怪兽需要面对左侧；战斗双方必须面对面。
- UI 图集里的按钮、难度、开始战斗、数字键等长方形按钮必须用 Phaser `nineslice`，不要直接 `setDisplaySize` 拉伸整张按钮图。静音按钮当前用 Phaser 圆形绘制是有意为之，因为 UI ring 切图边缘不够干净。
- 奥特曼选择卡片里的强度用完整星级图标表示，不显示原始攻击/防御/生命数值。不要复用会被裁切的 `spark-gold` 小图作为星级。

## 战斗核心逻辑

- 战斗题目显示要大、居中、清楚；孩子只按数字键，按够正确答案位数后自动判定并发射。不要增加退格、清空或发射按钮。
- 战斗倒计时不能只是单色细条；必须保留外框、暗槽、发光填充、末端光点和时间不足时的黄/红警示状态。
- 答对触发奥特曼攻击；答错或超时触发怪兽攻击；连续答对 3 题触发绝招，连对数清零。
- 奥特曼/怪兽语音属于本地 SFX，必须通过 `playSfx` 触发，从而遵守全局静音；不要直接绕过静音状态播放角色语音。
- 噩梦/地狱对应当前数据里的 `legend`/`ultimate`，通过血量、攻防、奖励、限时和额外项数提升难度，不要把难度标签和内部 id 混淆。
- 胜利只在怪兽血量归零后进入结算；怪兽被击败必须先播放爆炸和震动，再展示胜利结算。

## 动效时序

- 普通攻击、受击、绝招、爆炸、结算的播放时间要让孩子能看清，不能为了响应快而把关键画面一闪而过。
- 绝招是全屏蒙太奇：全屏暗场/闪光/斜向光带/奥特曼剪影/绝招名称大字都应覆盖整个画布。绝招光线、命中特效和爆炸的深度必须高于蒙太奇遮罩，不能被中间遮住。
- 奥特曼光线特效使用前端粒子和几何光效实现，普通攻击至少有光束、粒子、命中环，绝招必须额外包含旋涡粒子、十字星闪光和圆形冲击波；光线坐标应穿过角色身体之间，不能和倒计时条重叠，也不要退化成单条静态矩形或静态位图。
- 绝招名称不能只用 ADD 发光文字直接压在题目上；必须有暗底和稳定的非 ADD 标题层，发光层只能做装饰，避免算式从字形中透出来。
- 转场应是全屏扫光或全屏遮罩淡出，不能只在屏幕中间飞两张小切图。
- 结算页必须有稳定的 Phaser 文字作为主信息，结果图片只能作为装饰或增强。不要让胜利/失败、奖励、按钮互相遮挡。

## 验证流程

- 代码改完先运行 `npm run build`。
- 前端视觉改动必须打开本地 Vite 页面验证，当前常用地址是 `http://127.0.0.1:5175/`。
- 至少截图检查：首页背景、故事模式背景和奥特曼卡片星级、战斗题目位置、绝招全屏效果、怪兽爆炸、胜利/失败结算。
- 验证截图时重点看：背景是否空白、文字是否越界、按钮切图是否错位、数字键是否裁切、sprite 是否有灰底、角色帧切换是否上下错位、重要动效是否被遮罩挡住。
